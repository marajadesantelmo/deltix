Proyecto de Mejora del Chatbot Deltix

Estado actual

El chatbot Deltix está desarrollado utilizando el módulo de Telegram para Python, lo que restringe su uso exclusivamente a Telegram. Esto limita su alcance, especialmente en el Delta del Tigre, donde WhatsApp es la plataforma de mensajería predominante. La gestión de datos, como experiencias de usuario y suscripciones, se realiza mediante archivos .csv, una solución subóptima para el manejo de grandes volúmenes de información y escalabilidad. El proyecto se encuentra alojado en PythonAnywhere, una plataforma adecuada para proyectos iniciales, pero que presenta limitaciones en comparación con una solución basada en una máquina virtual (VM) con bases de datos robustas como PostgreSQL. Por último, la interacción del chatbot está basada en reglas de expresiones regulares (regex), lo que restringe la fluidez y la capacidad de respuesta del sistema. Esta limitación podría abordarse mediante la implementación de un prompt engineering de bajo costo utilizando el servicio de OpenAI, lo que permitiría interacciones más inteligentes y contextuales.

Para mejorar Deltix y alcanzar su máximo potencial, se propone un proyecto dividido en tres etapas, con un esfuerzo estimado de 160 horas por etapa (1 mes de trabajo full-time por etapa).

Etapa 1: Ampliación de Infraestructura y Compatibilidad Multiplataforma

Objetivo: Migrar a una arquitectura más robusta, escalable y flexible, habilitando la interacción con múltiples plataformas.

  • Desarrollo de una API personalizada:
    • Crear una API RESTful utilizando Flask (Python) para centralizar las funcionalidades del chatbot.
    • La API permitirá la integración con diferentes plataformas como WhatsApp, Telegram y el servicio de OpenAI.
  • Migración a PostgreSQL:
    • Sustituir la gestión de datos en archivos .csv por una base de datos relacional PostgreSQL.
    • Mejorar la eficiencia, escalabilidad y seguridad de la gestión de información.
  • Cambio de alojamiento:
    • Migrar el proyecto desde PythonAnywhere a una máquina virtual (VM) en servicios como AWS, Google Cloud o Azure, con soporte para PostgreSQL.
  • Integración con WhatsApp Business Platform:
    • Implementar compatibilidad con WhatsApp Business Platform API para ampliar el alcance del chatbot hacia la plataforma más utilizada en la región.
  • Rediseño de la interacción en Telegram:
    • Reconstruir la funcionalidad del chatbot en Telegram utilizando la nueva API centralizada.

Etapa 2: Implementación de Prompt Engineering y Mejora de Interacciones

Objetivo: Incorporar inteligencia contextual para mejorar la interacción con los usuarios.

  • Desarrollo de prompt engineering:
    • Integrar la API de OpenAI para permitir interacciones más naturales e inteligentes.
    • Diseñar prompts que optimicen la precisión y relevancia de las respuestas del chatbot, manteniendo bajos costos operativos.

Etapa 3: Funcionalidades Avanzadas y Plataforma de Gestión

Objetivo: Expandir las capacidades del chatbot y ofrecer valor agregado a los usuarios.

  • Funcionalidad de "Voy y Vuelvo":
    • Permitir a los usuarios compartir viajes en embarcaciones privadas desde y hacia las islas.
    • Incorporar una interfaz intuitiva para gestionar solicitudes y ofertas de transporte.
  • Guía de productos y servicios locales:
    • Desarrollar una funcionalidad para listar productos y servicios ofrecidos en el Delta.
    • Incluir filtros y categorías para una búsqueda rápida y eficiente.
  • Content Management Service (CMS):
    • Implementar un CMS en PHP para que los usuarios puedan agregar, editar o eliminar información sobre productos y servicios.
    • Diseñar una interfaz web responsiva y amigable para dispositivos móviles.

Entregables por Etapa

  • Etapa 1:
    • API RESTful de Deltix implementada en Flask.
    • Migración de datos a PostgreSQL y eliminación del uso de archivos .csv.
    • Proyecto migrado a una máquina virtual con configuración de base de datos.
    • Integración funcional con WhatsApp Business Platform.
    • Rediseño de la interacción con Telegram utilizando la nueva API.
  • Etapa 2:
    • Prompt engineering configurado con OpenAI.
    • Respuestas del chatbot optimizadas para mayor precisión y adaptabilidad al contexto.
  • Etapa 3:
    • Funcionalidad de "Voy y Vuelvo" operativa.
    • Sistema de guía de productos y servicios locales completamente funcional.
    • CMS implementado con interfaz responsiva para autogestión de contenido por parte de los usuarios.
