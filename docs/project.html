Improvement Project for the Deltix Chatbot

Current State

The Deltix chatbot is developed using the Telegram module for Python, limiting its use exclusively to Telegram. This restricts its reach, especially in the Tigre Delta, where WhatsApp is the dominant messaging platform. Data management, such as user experiences and subscriptions, is done through .csv files, which is a suboptimal solution for handling large volumes of data and scalability. The project is hosted on PythonAnywhere, a platform suitable for initial projects but with limitations compared to a virtual machine (VM)-based solution with robust databases like PostgreSQL. Lastly, the chatbot's interactions are based on regular expression (regex) rules, which restrict the system's fluency and responsiveness. This limitation could be addressed by implementing low-cost prompt engineering using OpenAI's service, enabling more intelligent and contextual interactions.

To enhance Deltix and unlock its full potential, the project proposes three stages, with an estimated effort of 160 hours per stage (1 month of full-time work per stage).

Stage 1: Infrastructure Expansion and Multi-Platform Compatibility

Objective: Migrate to a more robust, scalable, and flexible architecture, enabling interaction with multiple platforms.

  • Development of a Custom API:
    • Create a RESTful API using Flask (Python) to centralize chatbot functionalities.
    • The API will enable integration with various platforms such as WhatsApp, Telegram, and OpenAI's service.
  • Migration to PostgreSQL:
    • Replace data management in .csv files with a PostgreSQL relational database.
    • Improve data handling efficiency, scalability, and security.
  • Hosting Migration:
    • Migrate the project from PythonAnywhere to a virtual machine (VM) with PostgreSQL support.
  • Integration with WhatsApp Business Platform:
    • Implement compatibility with the WhatsApp Business Platform API to extend the chatbot's reach to the region's most-used platform.
  • Redesign of Telegram Interaction:
    • Rebuild the chatbot functionality on Telegram using the new centralized API.

Stage 2: Prompt Engineering Implementation and Interaction Enhancement

Objective: Incorporate contextual intelligence to improve user interactions.

  • Development of Prompt Engineering:
    • Integrate the OpenAI API to enable more natural and intelligent interactions.
    • Design prompts to optimize the chatbot's response accuracy and relevance while keeping operational costs low.

Stage 3: Advanced Features and Management Platform

Objective: Expand the chatbot's capabilities and provide added value to users.

  • "Voy y Vuelvo" Feature:
    • Enable users to share private boat trips to and from the islands.
    • Incorporate an intuitive interface to manage transportation requests and offers.
  • Local Products and Services Guide:
    • Develop a feature to list products and services available in the Delta.
    • Include filters and categories for quick and efficient searches.
  • Content Management Service (CMS):
    • Implement a CMS in PHP for users to add, edit, or remove information about products and services.
    • Design a responsive and mobile-friendly web interface.

Deliverables per Stage

  • Stage 1:
    • RESTful API for Deltix implemented in Flask.
    • Data migrated to PostgreSQL, eliminating the use of .csv files.
    • Project migrated to a virtual machine with database configuration.
    • Functional integration with WhatsApp Business Platform.
    • Redesigned interaction with Telegram using the new API.
  • Stage 2:
    • Prompt engineering configured with OpenAI.
    • Chatbot responses optimized for greater accuracy and context adaptability.
  • Stage 3:
    • "Voy y Vuelvo" functionality operational.
    • Fully functional local products and services guide system.
    • CMS implemented with a responsive interface for user-managed content.
